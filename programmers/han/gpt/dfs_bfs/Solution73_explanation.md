# 2배 스케일링이 필요한 이유

## 문제 상황

테스트 케이스 1의 사각형들:
- 사각형1: (1,1) - (7,4)
- 사각형2: (3,2) - (5,5)  
- 사각형3: (4,3) - (6,9)
- 사각형4: (2,6) - (8,8)

## 원본 좌표계 (1배)의 문제점

### 예시 1: 모서리에서 만나는 경우
```
사각형 A: (1,1) - (3,3)
사각형 B: (3,3) - (5,5)

(3,3)에서 두 사각형이 모서리로만 만납니다.

원본 좌표계:
(3,3) → (4,3)로 이동할 때
- (4,3)은 사각형 B의 테두리입니다
- 하지만 (3,3) → (4,3) 경로가 
  사각형 내부를 통과하는지 테두리를 따라가는지
  구분이 어렵습니다!
```

### 예시 2: 대각선으로 인접한 경우
```
사각형 A: (1,1) - (3,3)
사각형 B: (2,2) - (4,4)

겹치는 영역: (2,2) - (3,3)

원본 좌표계에서 (2,2) → (3,3)으로 가려면:
- 경로1: (2,2) → (3,2) → (3,3) [테두리 경로]
- 경로2: (2,2) → (2,3) → (3,3) [테두리 경로]

하지만 BFS가 (2,2)에서 (3,3)으로 바로 가는 것처럼
보일 수 있습니다 (대각선 이동은 아니지만).
```

## 2배 스케일링의 해결책

### 2배로 확대하면:
```
사각형 A: (2,2) - (6,6)
사각형 B: (6,6) - (10,10)

이제 (6,6)에서 (7,6)으로 가는 경로가 명확합니다:
- (6,6) → (7,6) → (8,6) → ... (테두리 경로)
- 내부를 통과하는 경로는 불가능합니다!
```

### 왜?
1. **테두리와 내부의 명확한 구분**
   - 테두리: 짝수 좌표 (또는 경계)
   - 내부: 홀수 좌표 (테두리 안쪽)
   
2. **인접 사각형 사이의 빈 공간 생성**
   - 원본: (3,3)에서 (4,3)으로 바로 이동 가능
   - 2배: (6,6)에서 (7,6)으로 가는데, 
          (7,6)이 정확히 어느 사각형의 테두리인지 명확

3. **내부 제거의 정확성**
   ```java
   // 내부 제거: x1+1 ~ x2-1, y1+1 ~ y2-1
   // 2배 좌표계에서는 이 범위가 정확히 내부만 제거
   ```

## 실제 문제에서 발생하는 경우

테스트 케이스 1에서:
- 여러 사각형이 겹치고 모서리에서 만남
- (4,3) 지점: 사각형1, 2, 3이 모두 만나는 지점
- 원본 좌표계에서는 이 지점에서 잘못된 경로 선택 가능
- 2배 스케일링으로 정확한 테두리 경로만 선택 가능

## 결론

2배 스케일링은 "꼼수"가 아니라 **필수**입니다!
- 사각형의 테두리와 내부를 명확히 구분
- 대각선으로 인접한 사각형들 사이의 경로 문제 해결
- BFS가 정확한 테두리 경로만 따라가도록 보장
